# PRD 评审 Agent

一个多角色 PRD 评审 Agent，模拟产品经理、技术专家、市场专家和高管四个角色对产品需求文档进行全面评审。

## 功能特点

- 🎯 **四个专业角色**：产品经理、技术专家、市场专家、高管
- 📊 **综合评分**：1-5 分评分系统，基于多维度评估
- 📝 **结构化报告**：包含优势、风险、建议和问题的详细反馈
- ✅ **Go/No-Go 决策**：明确的推进建议和理由
- 📋 **优先级行动项**：按优先级分类的可执行建议
- 🔍 **参考文档支持**：可结合公司战略、技术指南等参考文档进行评审

## 使用方法

### 基础用法

直接提供 PRD 文档进行评审：

```
请使用 PRD 评审 Agent 评审这个 PRD：

[粘贴你的 PRD 内容]
```

### 带参考文档的评审

为了获得更有针对性的评审，可以提供参考文档（如公司战略、技术指南、市场研究等）：

```
请使用 PRD 评审 Agent 评审这个 PRD。

## PRD 文档
[你的 PRD 内容]

## 参考文档

### 公司战略 2026
[战略文档内容]

### 技术架构指南
[架构指南内容]
```

### 聚焦特定方面

你也可以要求重点关注特定方面：

```
请使用 PRD 评审 Agent 重点关注技术可行性和市场机会来评审这个 PRD：

[PRD 内容]
```

## 评审输出

Agent 会生成一份结构化的评审报告，包含以下部分：

### 1. 执行摘要
- 综合评分（1-5 分）
- Go/No-Go/有条件 Go 的推荐
- 关键亮点
- 关键问题（如有）

### 2. 四个角色的详细评审

每个角色（产品经理、技术专家、市场专家、高管）都会提供：
- **优势**：做得好的方面
- **风险**：潜在问题和担忧
- **建议**：具体的改进建议
- **问题**：需要澄清的问题

### 3. 优先级行动项
- 高优先级：必须解决的关键问题
- 中优先级：应该改进的重要事项
- 低优先级：可选的增强建议

### 4. 最终决策
- 明确的 Go/No-Go 决策
- 详细的决策理由
- 条件（如果是有条件 Go）
- 下一步行动

## 评分系统

**5 分 - 优秀**：PRD 全面、结构良好，可以推进。风险最小且可管理。

**4 分 - 良好**：PRD 扎实，有小的差距。需要小的调整，风险可管理。

**3 分 - 一般**：PRD 有明显差距需要解决。需要改进，风险中等到高。

**2 分 - 较差**：PRD 有重大问题。需要大量返工，风险高。

**1 分 - 严重**：PRD 存在根本性缺陷。不要推进。

## 评审维度

Agent 从四个维度评估 PRD，每个维度权重相等（25%）：

1. **产品市场契合度**（25%）
   - 用户问题清晰度和验证
   - 解决方案适当性
   - 功能优先级
   - 成功指标质量

2. **技术可行性**（25%）
   - 技术可实现性
   - 架构合理性
   - 实施复杂度
   - 风险水平

3. **市场可行性**（25%）
   - 市场机会规模
   - 竞争定位
   - 定价策略
   - GTM 方法

4. **商业价值**（25%）
   - 预期商业影响
   - ROI 潜力
   - 战略一致性
   - 资源效率

## 最佳实践

### 准备 PRD 时
- 包含清晰的问题陈述和用户验证
- 定义具体的成功指标（KPI）
- 说明技术要求和约束
- 提供市场背景和竞争分析
- 阐明商业目标和预期成果

### 提供参考文档时
- 公司战略文档（帮助评估战略一致性）
- 技术架构指南（帮助评估技术可行性）
- 市场研究报告（帮助评估市场机会）
- 现有产品文档（帮助评估集成和一致性）

### 使用评审结果
- 优先处理高优先级行动项
- 回答所有角色提出的问题
- 考虑所有建议并决定是否采纳
- 根据反馈迭代 PRD
- 必要时进行第二轮评审

## 故障排除

### PRD 信息不足
如果 PRD 缺少关键部分，Agent 会：
- 发出警告，说明缺少哪些部分
- 基于可用信息进行部分评审
- 在"问题"部分标记差距

### 没有参考文档
如果没有提供参考文档，Agent 会：
- 基于一般最佳实践和行业标准进行评审
- 在执行摘要中注明这一点

### 专业领域 PRD
对于高度专业化的领域（如 AI/ML、区块链、医疗设备），Agent 会：
- 应用通用的产品、商业和技术原则
- 标记可能需要领域专家评审的方面
- 诚实说明某些方面超出一般专业知识范围
