# PRD 评审 Agent 套件

一个多角色 PRD 评审工具集，包含 4 个独立的专业评审 Agent 和 1 个协调器，模拟真实的多方评审流程。

## 功能特点

- 🎯 **4 个独立的专业 Agent**：
  - **产品经理** - 用户需求、产品战略、功能优先级、KPI
  - **技术专家** - 技术可行性、架构设计、性能安全、工作量
  - **市场专家** - 市场机会、竞争分析、定价策略、GTM
  - **高管** - 商业价值、ROI、战略对齐、Go/No-Go 决策

- 🔄 **协调器 Agent** - 汇总所有评审，生成综合报告

- 📊 **独立评分系统** - 每个角色独立评分（1-5 分）

- 📝 **结构化报告** - 每个角色提供优势、风险、建议和问题

- ✅ **灵活使用** - 可以单独调用某个角色，或依次调用所有角色

## 安装

### 在 Claude Code 中安装

1. **添加 marketplace**：
```bash
/plugin marketplace add meggie1997/prd-review-agent
```

2. **安装整个套件**：
```bash
/plugin install prd-review-suite@meggie1997/prd-review-agent
```

这将安装所有 5 个 skills：
- `prd-review-pm` - 产品经理
- `prd-review-tech` - 技术专家
- `prd-review-market` - 市场专家
- `prd-review-exec` - 高管
- `prd-review-coordinator` - 协调器

## 使用方法

### 方式 1：单独调用某个角色

如果你只想要某个特定角色的评审：

```
请使用产品经理视角评审这个 PRD：

[粘贴 PRD 内容]
```

或者：

```
请使用技术专家视角评审这个 PRD：

[粘贴 PRD 内容]
```

### 方式 2：依次调用所有角色

获得完整的多角色评审：

**步骤 1 - 产品经理评审**：
```
请使用产品经理视角评审这个 PRD：

[粘贴 PRD 内容]
```

**步骤 2 - 技术专家评审**：
```
请使用技术专家视角评审这个 PRD：

[粘贴 PRD 内容]
```

**步骤 3 - 市场专家评审**：
```
请使用市场专家视角评审这个 PRD：

[粘贴 PRD 内容]
```

**步骤 4 - 高管评审**：
```
请使用高管视角评审这个 PRD 并给出 Go/No-Go 决策：

[粘贴 PRD 内容]
```

**步骤 5 - 汇总评审**：
```
请使用 PRD 评审协调器汇总以下评审结果：

## 产品经理评审
[粘贴产品经理的评审报告]

## 技术专家评审
[粘贴技术专家的评审报告]

## 市场专家评审
[粘贴市场专家的评审报告]

## 高管评审
[粘贴高管的评审报告]
```

### 方式 3：带参考文档的评审

为了获得更有针对性的评审，可以提供参考文档：

```
请使用产品经理视角评审这个 PRD。

## PRD 文档
[你的 PRD 内容]

## 参考文档

### 公司产品战略
[战略文档内容]
```

## 各角色评审重点

### 产品经理 (prd-review-pm)
- ✅ 用户需求清晰度和验证
- ✅ 解决方案适当性
- ✅ 功能优先级框架
- ✅ KPI 的 SMART 性
- ✅ 用户体验考虑

**输出**：优势、风险、建议、问题 + 5 个维度评分

### 技术专家 (prd-review-tech)
- ✅ 技术可行性评估
- ✅ 架构和设计考虑
- ✅ 性能和可扩展性
- ✅ 安全和合规要求
- ✅ 实施复杂度（T-shirt sizing）

**输出**：优势、风险、建议、问题 + 5 个维度评分 + 工作量估算

### 市场专家 (prd-review-market)
- ✅ 市场机会规模（TAM/SAM/SOM）
- ✅ 竞争格局分析
- ✅ 定价策略评估
- ✅ GTM 策略
- ✅ 客户获取可行性

**输出**：优势、风险、建议、问题 + 5 个维度评分 + 市场分析

### 高管 (prd-review-exec)
- ✅ 商业价值主张
- ✅ ROI 和财务分析
- ✅ 战略一致性
- ✅ 资源分配合理性
- ✅ 风险 vs 回报评估

**输出**：优势、风险、建议 + 5 个维度评分 + Go/No-Go 决策

### 协调器 (prd-review-coordinator)
- ✅ 汇总所有角色评审
- ✅ 识别共识和分歧
- ✅ 综合评分计算
- ✅ 优先级行动项
- ✅ 最终推荐决策

**输出**：综合评审报告 + 综合评分 + 最终推荐

## 评分系统

每个角色独立评分（1-5 分）：

- **5 分 - 优秀**：该维度表现出色，无重大问题
- **4 分 - 良好**：表现良好，有小的改进空间
- **3 分 - 一般**：有明显不足，需要改进
- **2 分 - 较差**：存在重大问题，需要大幅修改
- **1 分 - 严重**：基本要素缺失，需要重新设计

协调器会计算综合评分：
```
综合评分 = (产品经理评分 + 技术专家评分 + 市场专家评分 + 高管评分) / 4
```

## 使用场景

### 场景 1：快速单一视角评审
只需要某个特定角色的意见时，直接调用对应的 Agent。

**适用于**：
- 技术可行性快速评估
- 市场机会初步判断
- 产品方向验证

### 场景 2：完整多方评审
需要全面评审时，依次调用所有 4 个角色，最后用协调器汇总。

**适用于**：
- 重要产品决策
- 大型项目立项
- 战略级产品规划

### 场景 3：迭代评审
PRD 修改后，只重新评审相关角色。

**适用于**：
- 技术方案调整后重新评审技术可行性
- 定价策略修改后重新评审市场视角

## 最佳实践

### 准备 PRD 时
- 包含清晰的问题陈述和用户验证
- 定义具体的成功指标（KPI）
- 说明技术要求和约束
- 提供市场背景和竞争分析
- 阐明商业目标和预期成果

### 提供参考文档时
- **产品经理**：产品战略、用户研究报告
- **技术专家**：技术架构指南、技术栈标准
- **市场专家**：市场研究报告、竞争分析
- **高管**：公司战略、OKR、财务目标

### 使用评审结果
1. 先阅读每个角色的独立评审
2. 识别共同关注的问题（多个角色都提到）
3. 优先处理高优先级行动项
4. 回答所有角色提出的问题
5. 根据反馈迭代 PRD
6. 必要时进行第二轮评审


## 故障排除

### 某个角色的评审不够详细
- 提供更多的参考文档
- 在 PRD 中补充该角色关注的信息
- 明确要求该角色关注特定方面

### 不同角色的评审有矛盾
- 这是正常的！不同角色有不同的关注点
- 使用协调器来识别和分析这些分歧
- 根据公司优先级权衡不同意见

### 协调器汇总不完整
- 确保提供了所有 4 个角色的完整评审报告
- 检查每个评审报告是否包含所有必需部分
